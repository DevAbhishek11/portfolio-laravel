# May 19, 2026 — Portfolio Refactor & Feature Pass

This change log documents every modification made in this session. The goal
was to (1) clean up filter URLs, (2) fix a broken "Read More" link on the
public blog grid, (3) make every list page (frontend + admin) load
without a full page reload, (4) flesh out the admin panel with proper
show / comments / toggle screens, and (5) display per-blog comment
counts in the admin.

## 2. "Read More" Bug — Fixed

### Root cause

`.anime-card` had a `::before` pseudo-element with
`position: absolute; inset: 0; z-index: 0`. Because that pseudo-element
established a real stacking layer **above** the card's auto-z-index
children, it was capturing every click in the card body — including the
"Read →" link on the normal blog cards. The featured card worked only
because `.btn-anime` has its own `position: relative` which creates a
new stacking context above the pseudo-element.

### Fix

`resources/views/layouts/app.blade.php` — added `pointer-events: none`
to `.anime-card::before` so the hover-gradient overlay never intercepts
clicks. As a defense in depth, the new card partials also wrap text
inner content in a `position:relative; z-index:1;` container and the
Read More / View buttons themselves set `z-index: 2`, so even on cards
that re-introduce a stacking context the link wins.

```css
.anime-card::before {
    /* …existing… */
    pointer-events: none;
}
```

## 3. AJAX Everywhere — No Full Page Reloads on Filter / Sort / Paginate

Implemented with a small inline IIFE per page. No new build step, no
extra library. Each script:

1. Reads initial state from the rendered Blade props.
2. Builds two URLs on every change: a "real" clean URL for
   `history.pushState` (so the address bar still shows
   `/blogs/category/Laravel?sort=popular`) and an AJAX URL pointing
   at the JSON endpoint.
3. Fetches the JSON, swaps the `innerHTML` of a single wrapper, and
   re-binds delegated handlers via the wrapper itself (so pagination
   links inserted by the AJAX response keep working).
4. Hooks `popstate` so the browser back / forward buttons re-render
   the matching state without a full reload.

### Frontend

- `/blogs` — search (debounced 350 ms), sort, category chips, tag chips,
  pagination — all AJAX. Clean URL in the address bar updates live.
- `/projects` — search, sort, category chips, pagination — all AJAX.
- `/blogs/{slug}` — comment form unchanged (it's a POST on a single
  blog, not a list). Tag chips point at the new clean URLs.

### Admin

- `/admin/blogs` — search, status filter, pagination — all AJAX.
- `/admin/blogs/{id}/comments` — search, visible-only filter,
  pagination, plus per-row "show on site" toggle and delete — all
  AJAX, no reload.
- `/admin/projects` — search, status, category, pagination — all AJAX.
- `/admin/contacts` — search, status, pagination — all AJAX. Bulk-action
  select-all logic re-wires itself after each AJAX swap.

### New partial files (used by AJAX swap)

- `resources/views/admin/blogs/_table.blade.php`
- `resources/views/admin/blogs/_comments_table.blade.php`
- `resources/views/admin/projects/_table.blade.php`
- `resources/views/admin/contacts/_table.blade.php`

## 5. Comment Review Pages (Admin)

New surface: `/admin/blogs/{id}/comments`.

### What it does

- Lists every comment on the blog (approved + unapproved + replies).
- Per-row toggle switch: **Visible / Hidden**. Flipping it hits
  `POST /admin/blogs/{id}/comments/{commentId}/toggle`, which inverts
  `is_approved` and returns JSON. The label flips in place, no reload.
- Per-row delete button (DELETE, AJAX).
- Search across name, email, comment body.
- Filter dropdown: All / Visible only / Hidden only.
- Pagination — AJAX.

### Controller changes — `app/Http/Controllers/Admin/BlogController.php`

- Added `comments()` (full page) and `commentsAjax()` (JSON).
- Added `toggleComment()` — flips `is_approved` and returns JSON when
  called via XHR.
- Existing `approveComment()` and `deleteComment()` were updated to
  return JSON for XHR callers.
- `index()` now `withCount(['comments', 'approvedComments'])` so the
  list table can show per-blog comment counts cheaply.

### Visibility = `is_approved`

The frontend uses the existing `approvedComments()` relation to decide
what to render on the public blog page. Toggling `is_approved` from the
admin therefore directly controls whether a comment appears on the
public site — no schema change needed.

## 7. Admin Pagination — No Reload

Every admin list page now wraps its table in an `id="...-wrapper"`
container. Pagination links inside that wrapper are intercepted by a
delegated `click` handler that:

1. Reads the `?page=N` from the link's `href`.
2. Calls the matching `*.ajax` endpoint with the current filter state.
3. Replaces the wrapper's `innerHTML` with the response.
4. Pushes a clean URL into the address bar so the back button works.

This applies to: `admin/blogs`, `admin/blogs/{id}/comments`,
`admin/projects`, `admin/contacts`.

## 11. Notes / Follow-ups

- The existing `approveComment` endpoint is preserved for back-compat
  but is now functionally a subset of the new `toggleComment` — both
  routes coexist; you can remove `approveComment` later if nothing
  external depends on it.
- The frontend AJAX scripts use `history.pushState` only. Refreshing
  the page on a deep filter (e.g. `/blogs/category/Laravel?sort=popular`)
  works because the Laravel route resolves the same combination
  server-side.
- The Locomotive smooth-scroll wrapper is left untouched; the new
  partials are loaded into a container inside the existing
  `[data-scroll-container]`, so smooth scroll keeps working.
- Spatie Media Library is still the source of truth for project
  thumbnails on the public site; the legacy `thumbnail` column is
  used as a fallback in the new `_grid.blade.php` partial so existing
  projects keep rendering.
